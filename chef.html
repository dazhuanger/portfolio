Chef

The objective of this design exercise is to use lean UX principles to design the user interface of an Android mobile application and a Wear OS application.

Design Rationales

As a cooking companion App, Chef should provide the user with some basic features such as step-by-step recipes and timer. Also, it is important to add some discoverability to this app by offering users recipe recommendations and a searching feature to find recipes according to their liking.

Mid-Fi Wireframe

Overall Structure

  • Bottom navigation bar

    • Organizes different pages and allows the user to navigate inside the app.

  • Home page

    • Contains recipe recommendations and the user's browsing history. The page layout gives the user an affordance to scroll down and swipe to the left to explore more content.

  • Search page

    • Placed on the top of the screen where the user can type in the keywords to search a specific recipe. Also, the user can tap on any block below to open a category and view the recipes that he or she is interested in.

  • Favourite list

    • To save the favourite recipes.

  • Shopping list

    • A checklist of essential ingredients so that they can be fully prepared before cooking.

Recipe Page

  • Introduction

    • Including the name, type of cuisine and ​a gallery of the recipe.

  • Time and nutrition information

    • More detailed information for the user.

  • Ingredient list

    • The user can tap on the button on the top right corner to create a shopping list for the ingredients.

Steps Page

  • Timeline

    • Located on the top of the screen, indicating the current step. Also serves as an alternative to swiping gestures. Users can tap on the dot to skip multiple steps at a time.

  • Photo/Video and text instructions

    • Depending on the complexity of the recipe, the content maker can choose to put a video or a simple photo.

  • Timer

    • Users can tap on the button to set a timer. Once the timer is set, the countdown clock will replace the timer button.

Watch UI

  • Navigation

    • The navigation bar in the mobile UI is adjusted to accommodate the watch screen.

  • Text instructions

    • Users can only see text instructions on their watches due to the screen size restriction.

  • Timer

    • Same as the mobile UI, users can tap on the button to set a timer. Once the timer is set, the countdown clock will replace the timer button.

High-Fi Prototypes

Added Elements

  • Colours & Interactions

    • High-fi prototypes are created following Material Design guidelines.

  • Enhanced feedback

    • A pop-up that indicates the new shopping list is created. A blue dot on the shopping list icon in the bottom navigation bar to guide the user to the newly created shopping list.

  • Dimmed background

    • The dimmed background creates a stronger contrast which allows the user to focus on the current task such as setting a timer.
