Ticket Faster

The objective of this design exercise is to use lean UX principles to design the user interface of a desktop application for macOS.

Design Rationales

This desktop application is designed for a small independent theatre venue with a raised proscenium stage and 120 permanently fixed chairs. The MVP allows the box office attendant to:

  • See and select from a list of upcoming performances

  • See details about a selected performance

  • See tickets sold and seats remaining for a selected performance

  • Select seats to sell to a customer, see their combined price and initiate the sale

  • Confirm the sale and print tickets or e-mail them to the customer

Mid-Fi Wireframe

As an application for the box office attendant instead of audiences, it is important to keep the design lean and focus on the key functionalities instead of dazzling visual effects.

  • Split view

    • Increases information density.

  • Search bar

    • Located at the top left corner to provide searchability.

  • List of performances

    • Located on the left part of the split view. Each item on the list shows the date, time, name of the performance and seat availability.

  • Tabs

    • Located on top of the split view for the user to view the list of different dates. Two arrow buttons are provided to switch to next or previous week.

  • Seat map

    • Shows the seat availability.

  • Pop-up window

    • To confirm the ticket information and initiate a sale.

Feedback

"The tabs show today by default. What if the customer wants to order tickets for a performance a few weeks in advance? Maybe make the date searchable as well?"

"For the seat map, consider adding some annotations to tell which colour means what. And try not to use green and red because of the accessibility issue. The ticks on the icon of selected seats are hard to see, try to use another colour."

"It’s better to show the unit prices separately in the check-out window. Is there any way that I can see the unit price of each seat before checking out? Some customer may want to select seats according to the price."

High-Fi Mockup

Based on the feedback on the mid-fidelity wireframe, several features are improved in the high-fidelity prototype.

Improved Features

  • A date selector to search by date
  • Annotations under the seat map
  • A popover to see the unit price of an individual seat
